Combined QA Methods
2/3 majority QA flags are required to exclude a snow pillow from being used in the MLR Prediction model (i.e., set to NaN).
Static QA
Uses absolute SWE and peer-median comparisons (no regression) to flag suspect data:
  • Hard checks: Flags missing data (NaN), physically impossible values (SWE < 0 or > 3000 mm), and NaN-to-value transitions (previous day NaN, current day finite).
  • dSWE envelope: Flags daily SWE change that falls outside the 0.5th–99.5th percentile range of historic daily changes, scaled by an engineering buffer (1.5×).
  • Peer-median residual: Computes the median SWE of the top-5 most correlated pillows (by R²). Flags the target if its absolute residual from that peer median exceeds a MAD-based threshold (3σ × 1.2× SE buffer, with minimums of 25 mm absolute and 5% relative).
  • Melt gating: Skips the peer-median check when both the current pillow and peer median are below a snow-present threshold (50 mm) to avoid false positives during melt-out.
Voting QA
Each correlated pillow "votes" on whether the target's daily behavior is consistent, using pairwise OLS regressions on daily dSWE:
  • Hard checks: Same as Static QA — missing data, physical range, NaN-to-value transition.
  • dSWE envelope: Same quantile-based envelope check as Static QA.
  • Peer-regression voting: For each of the top-5 dynamically ranked correlated pillows, fits a simple linear regression (dSWE_target ~ dSWE_voter) on historic data. Each voter casts a weighted "OK" or "suspect" vote based on whether the target's residual exceeds a MAD-based threshold. The pillow is flagged when the R²-weighted fraction of OK votes falls below 60%.
  • Trimming: The most extreme voter (by absolute residual) is trimmed before tallying, as long as the minimum voter count (3) is maintained.
  • Melt gating: Suppresses dSWE envelope and voting flags when both current and previous SWE are below the snow-present threshold (50 mm).
SnowModel QA
Uses HRRR-driven SnowModel gridded SWE as an independent reference to flag suspect data:
  • Hard checks: Same as Static and Voting — missing data, physical range, NaN-to-value transition.
  • dSWE envelope: Same quantile-based envelope check as Static QA.
  • SnowModel prediction interval: Uses OLS regression of observed SWE against HRRR SnowModel SWE at the 3 best-correlated grid cells (best, second, third) with seasonal sin/cos terms. Flags the pillow when |observed − predicted| exceeds a threshold based on the prediction standard error (z=2.0 × SE_buff=1.2 × SE_pred), with minimums of 25 mm absolute and 5% relative.
  • Melt gating: Suppresses dSWE envelope and prediction-residual flags when both observed and predicted SWE are below the snow-present threshold (50 mm).
