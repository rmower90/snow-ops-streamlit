Combined QA Methods
2/3 majority QA flags are required to exclude a snow pillow from being used in the MLR Prediction model (i.e., set to NaN). Generic data-integrity checks (missing data, physical-range violations) are computed by all three methods and produce deterministic agreement; method-specific checks (peer-residual, peer-vote, snowmodel-residual) are unique to each method and so the majority vote requires independent corroboration. A "Manual removed" overlay marks days/pillows that are excluded via the basin's YAML configuration regardless of automated QA.
Static QA
Uses absolute SWE and peer-median comparisons (no regression) to flag suspect data:
  • Hard checks: Flags missing data (NaN), physically impossible values (SWE < 0 or > 3000 mm), and NaN-to-value transitions (previous day NaN, current day finite).
  • dSWE envelope: Flags daily SWE change that falls outside the 0.5th–99.5th percentile range of historic daily changes, scaled by an engineering buffer (1.5×).
  • Peer-median residual: Computes the median SWE of the top-5 most correlated pillows (by R²). Flags the target if its absolute residual from that peer median exceeds a MAD-based threshold (3σ × 1.2× SE buffer, with minimums of 25 mm absolute and 5% relative).
  • Melt gating: Skips the peer-median check when both the current pillow and peer median are below a snow-present threshold (50 mm) to avoid false positives during melt-out.
Note: the dSWE envelope and NaN-to-value checks are owned exclusively by Static QA (they were previously duplicated in Voting and SnowModel; those duplicates produced deterministic 3-of-3 majority agreement and were removed).
Voting QA
Each correlated pillow "votes" on whether the target's daily behavior is consistent, using pairwise OLS regressions on daily dSWE:
  • Hard checks: Missing data and physical-range only. (dSWE envelope and NaN-to-value are not evaluated here — see Static QA.)
  • Peer-regression voting: For each of the top-5 dynamically ranked correlated pillows, fits a simple linear regression (dSWE_target ~ dSWE_voter) on historic data. Each voter casts a weighted "OK" or "suspect" vote based on whether the target's residual exceeds a MAD-based threshold. The pillow is flagged when the R²-weighted fraction of OK votes falls below 60%.
  • Trimming: The most extreme voter (by absolute residual) is trimmed before tallying, as long as the minimum voter count (3) is maintained.
  • Melt gating: Suppresses voting flags when both current and previous SWE are below the snow-present threshold (50 mm).
SnowModel QA
Uses HRRR-driven SnowModel gridded SWE as an independent reference to flag suspect data:
  • Hard checks: Missing data and physical-range only. (dSWE envelope and NaN-to-value are not evaluated here — see Static QA.)
  • SnowModel prediction interval: Uses OLS regression of observed SWE against HRRR SnowModel SWE at the 3 best-correlated grid cells (best, second, third). Flags the pillow when |observed − predicted| exceeds a threshold based on the prediction standard error (z=2.0 × SE_buff=1.2 × SE_pred), with minimums of 25 mm absolute and 5% relative.
  • Melt gating: Suppresses the prediction-residual check when the SnowModel prediction is below the snow-present threshold (50 mm) to avoid flagging lingering pillow snow when the model predicts near zero.
